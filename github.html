Github Notları

Temel Terimler

  1. repository : Kısa ismiyle repo, kodlarımızın saklandığı yerdir
  2. master : Depodaki ana proje sürümüdür
  3. branch : Git mantığı ile çalışır ve projedeki geliştirmelerimizi ağaç dalı mantığı ile ayrı dallarda saklayabilir, sonrasında master branch de birleştirebiliriz
  4. merge : branch lerde yapılan işlemlerin master de birleştirilmesidir.
  5. commit : Commit işlemi localdeki dosyalarımızı depoya gönderilmek üzere hazırlanır
  6. push : Dosyalarımızı depoya göndermemizi sağlar
  7. pull : Depodaki dosyalarımızı çekmemizi sağlar
  8. .gitignore : Git in takip etmesini istemediğimiz dosya ve klasörleri belirttiğimiz dosyadır.

Vs code da projede bir değişiklik yapıldıktan sonra bunu github a göndermek için source control tagını açtık. Message alanına yapılan değişiklikle alakalı bir mesaj yazdık. Son olarak commit butonuna basıp depoya gönderilmek üzere hazırladık ce sync butonuna push işlemi yaparak değişikliği githuba yolladık.

Temel Git Komutları

  1. git add veya add *, git add index.html : Klasöre eklenen dosyaları gönderilmek üzere sahneye alma (stage)
  2. git status : klasördeki tüm değişiklikleri göster
  3. git commit -m "İlk commit" : Depoya gönderilmek için commiit işlemi yapılır
  4. git commit -a -m "Mesajımız" : Staging e ekle ve commitle
  5. git push : değişiklikleri depoya gönderme komutu
  6. git pull : Uzak depodaki depodaki dosyaları lokale indirme

Githuba yüklediğimiz projenin gizlilik ayarını github daki proje içerisine girip > settings den ayarları açıp > en alta indik ordan change visibility butonuna basarak private ceya public yapabiliriz.
